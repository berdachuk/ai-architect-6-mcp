# DEC-012 — Multi-agent-safe memory bank

| Field | Value |
|---|---|
| Status | Accepted |

## Problem

The original memory bank (DEC-006) used a handful of shared mutable Markdown files:

- `activeContext.md` — rewritten every task ("Current focus")
- `progress.md` — appended near a trailing milestone-status footer
- `decisions.md` — appended `DEC-###` blocks with ad-hoc numbering
- `plans/00-index.md` — in-place Active→Archived status edits

Under parallel multi-agent work (separate git worktrees), these become conflict hotspots: every agent rewrites the same files.

## Solution

Partition the memory bank into tiers by edit frequency and conflict risk:

| Tier | Files | Write pattern |
|---|---|---|
| Reference (hand-edited, rare) | `projectbrief.md`, `systemPatterns.md`, `techContext.md`, `productContext.md` (prose) | Direct edit; low conflict probability |
| Generated indexes (read-only to agents) | `activeContext.md`, `progress.md`, `decisions.md`, `productContext.md` tables, `plans/00-index.md` | Regenerated by `scripts/sync-memory-index.sh` from registries + records |
| Append-only registries (multi-agent safe) | `registry/*.jsonl` | One JSON line per ID; mint `max+1`; last-line conflicts trivially resolvable |
| Per-record files (append-only) | `records/progress/M{NN}.md`, `records/active/M{NN}.md`, `records/decisions/DEC-###.md`, `records/deferred/M{NN}.md` | One file per record; distinct files = zero merge conflict |
| Module locks (advisory) | `locks/<module>.md` | Acquire before editing coupled file pairs; release on merge |
| Worktree scratchpads (git-ignored) | `worktrees/<branch-slug>/` | Transient per-branch state; never merge to main |

## Conflict-prevention principles

- **Append, never rewrite.** Registries and record files are append-only. Editing an existing line breaks ID stability.
- **One record per file.** Two agents completing different milestones create distinct `records/progress/M{NN}.md` files.
- **Generated indexes are read-only to agents.** `sync-memory-index.sh --check` is a CI gate.
- **Acquire a module lock before editing coupled files** (e.g. prompt template + sanitizer).
- **Use worktree scratchpads for transient state** — never merge them.
- **Serialize, don't race.** If two agents need the same module, the second waits or coordinates.

## Migration (2026-06-21)

- `decisions.md` DEC-001..DEC-011 → `registry/dec.jsonl` (index) + `records/decisions/DEC-###.md` (body)
- `progress.md` entries → `records/progress/M{NN}.md` per milestone
- `activeContext.md` → generated from `records/active/*.md` + `registry/risk.jsonl` + `registry/scn.jsonl`
- Plans index → generated from `records/active/`, `records/deferred/`, `records/progress/`
